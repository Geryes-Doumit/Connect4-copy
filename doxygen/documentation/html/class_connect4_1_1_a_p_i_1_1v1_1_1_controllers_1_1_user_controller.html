async Task< IActionResult > Connect4.API.v1.Controllers.UserController.Login ( )

Endpoint to authenticate a user (login).

This endpoint expects an "Authorization" header using "Basic" encoding in Base64 format. The decoded format should be "username:password".

Possible HTTP status codes:

  • 200 OK - Valid credentials. Returns a success message and a JWT token.
  • 401 Unauthorized - Invalid username/password.
  • 400 Bad Request - Invalid input format (e.g., missing header, invalid base64, etc.).
  • 500 Internal Server Error - Unexpected internal error.
Returns
Returns a JSON object containing the LoginResponseDto if the login is successful. In case of an error, a JSON object containing the MessageResponsDto is returned.

◆ Logout()

IActionResult Connect4.API.v1.Controllers.UserController.Logout ( )

Endpoint to log out (invalidate the current JWT token).

This endpoint requires [Authorize], so a valid JWT must be supplied in the Authorization: Bearer <token> header.

If the token is valid, it retrieves the token's unique identifier (jti) and adds it to a blacklist. Any future requests using this token will be rejected (401).

Possible HTTP status codes:

  • 200 OK - Token successfully invalidated.
  • 401 Unauthorized - The token is invalid or not provided.
  • 400 Bad Request - The token is missing required claims (JTI, Exp).
  • 500 Internal Server Error - Unexpected internal error.
Returns
Returns a JSON object containing the MessageResponsDto with a message indicating the token has been invalidated. In case of an error, a JSON object containing the MessageResponsDto is returned.
